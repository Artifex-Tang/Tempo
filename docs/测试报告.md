### 6.2 调试过程中发现并修复的问题

| # | 问题现象 | 根因 | 修复 |
|---|---------|------|------|
| 1 | `automator.launch()` 报"Failed to launch" | Windows `child_process.spawn` 不能直接运行 `.bat` | 改用 HTTP `/v2/auto` + `automator.connect()` |
| 2 | DevTools HTTP 端口每次启动不同 | DevTools 2.x 动态分配端口 | 通过 `cli.bat islogin` 输出解析当前端口 |
| 3 | `navigateTo({url:'...'})` 报"url should be String not Object" | `changeRoute` 内部又包了一层 `{url:e}` | 直接传字符串：`navigateTo('/pages/...')` |
| 4 | 所有小程序网络请求失败 | `request.js` `BASE_URL=http://localhost:8080`，Docker 实际端口为 8081 | 改为 8081 |
| 5 | list API 返回空列表 | GET 请求把 `null` 参数序列化为 `?status=null`，Spring 解析 `Integer` 失败 400 | `request.js` GET 请求前过滤 null/undefined 参数 |
| 6 | `touristappid` 导致"好多请求失败" | tourist 模式限制 `wx.login()`，token 无法自动获取 | 重新导入项目时选"使用测试号" |
| 7 | `miniProgram.waitFor` 不存在 | `connect()` 模式不支持 `waitFor` | 改用 `sleep = ms => new Promise(r => setTimeout(r, ms))` |
| 8 | 异步 `callMethod` 不等 Promise | automator 对 async 页面方法只触发，不 await | 提交操作后通过 navigate 回来再进触发 `onShow → _loadData` |
