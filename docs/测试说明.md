## 六、T4 微信小程序仿真测试

### 6.1 环境准备

#### 第一步：安装微信开发者工具

下载稳定版（≥ 1.06.x）并安装。

#### 第二步：配置测试号 AppID

> **必须使用测试号**，`touristappid` 会限制 `wx.login()` 导致鉴权失败，所有 API 请求无 token。

1. 打开微信开发者工具
2. 点击左上角 **「...」→ 关闭项目**
3. 点击 **「+」→ 导入项目**
4. 目录选择 `Tempo/FocusLab`
5. AppID 行点击 **「使用测试号」**（DevTools 自动分配临时 appid，如 `wx5fb24a398538562a`）
6. 点击导入 → DevTools 自动更新 `project.config.json`

#### 第三步：开启服务端口

1. 菜单 **设置 → 安全设置**
2. 找到 **「服务端口」** → 点击开启

#### 第四步：关闭合法域名校验

DevTools **「详情 → 本地设置」** → 勾选：
- ✅ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书

#### 第五步：启动后端

```bash
cd Tempo
docker compose up -d
# 确认 http://localhost:8081/actuator/health 返回 {"status":"UP"}
```

#### 第六步：安装依赖并运行 E2E 测试

```bash
cd Tempo/FocusLab
npm install          # 首次安装 miniprogram-automator 等依赖
npm run test:e2e     # 运行仿真测试
```

> 测试启动时会自动：调用 CLI 发现 DevTools 端口 → HTTP `/v2/auto` 开启自动化 WebSocket → automator 连接 → 注入 JWT token → 执行用例。

## 八、已知约束

| 约束 | 说明 |
|------|------|
| AppID 必须用测试号 | `touristappid` 限制 `wx.login()`，导致 token 无法自动获取 |
| DevTools 服务端口每次启动变化 | launch-devtools.js 通过 CLI islogin 自动发现端口 |
| `callMethod` 对 async 方法不 await | 异步页面方法执行后需显式 sleep 或触发 onShow 等事件来等待数据 |
| GET 请求 null 参数 | WeChat 会将 null 序列化为字符串 "null"，request.js 已过滤 |
| Tab Bar 图标为占位图 | `FocusLab/images/` 下为 1×1px 占位 PNG，需替换为真实图标 |
